# 要件定義
- Nostr の CUI/TUI クライアント
    - 主機能
        - 投稿できる
        - イベントを受信できる。特にテキスト投稿
        - 複数リレーに接続できる
        - プロフィールが閲覧できる
- 初期実装としては、CUIクライアントとして実装する。後からTUI化する。
    - 投稿とタイムライン参照は別コマンドとなっても良い。
    - 後からTUI化しやすい構成を利用する。
- 下記の技術要件は、要件レベルで必須とする。
    - 開発言語は Go を中心に使う。
    - 稼働環境はGoシングルバイナリとする。
